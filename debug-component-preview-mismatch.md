# [OPEN] component-preview-mismatch

## 用户现象
- 用户希望打开“组件预览页”，但当前打开后看到的仍像“页面预览”。

## 期望
- 明确当前打开的到底是什么入口，并确认问题来自服务、地址、预览容器还是仓库结构本身。

## 初始假设
1. `8000` 端口不是当前仓库的静态服务。
2. IDE 预览容器与浏览器直开行为不同。
3. 当前组件预览入口实际就是根目录 `index.html` 文档页。
4. hash 或缓存导致没有停留在 `Form` 组件区。
5. 仓库没有独立的单组件预览页。

## 排查计划
1. 检查 `8000` 端口进程与根目录。
2. 直接抓取 `http://localhost:8000/` 返回内容。
3. 核对仓库内可用的 HTML 预览入口。
4. 对照结果给出根因与下一步方案。

## 运行时证据
- `lsof -nP -iTCP:8000 -sTCP:LISTEN` 显示 `node` 与 `Python` 同时监听 `8000`，其中 `node` 监听 `[::1]:8000`，`Python` 监听 `*:8000`。
- `curl http://localhost:8000/` 返回的是另一个 Vite 页面，标题为 `后台列表页 - 抖音来客`，不是当前仓库组件文档页。
- `curl http://127.0.0.1:8000/` 返回的是当前仓库的 `Life Design System - Components` 文档页。

## 结论
- 问题不是缓存。
- 根因是 `localhost` 优先命中了另一个 IPv6 的 Vite 进程，而不是当前项目的静态组件预览服务。
- 解决方式是改用 `127.0.0.1`，或换一个未冲突的新端口。
